## 🎯 Overview

ProLang Mini-Compiler is an educational project developed to demonstrate the fundamental concepts of compiler design. It implements the three main phases of a compiler:

1. **Lexical Analysis** - Tokenization of source code
2. **Syntactic Analysis** - Verification of grammatical structure
3. **Semantic Analysis & Code Generation** - Type checking and quadruplet generation

## ✨ Features

- ✅ **Complete Lexical Analysis** with FLEX (lex)
- ✅ **Robust Syntactic Analysis** with BISON (yacc)
- ✅ **Symbol Table Management** (Symbol Table - TS)
- ✅ **Intermediate Code Generation** (Quadruplets)
- ✅ **Type Checking**
- ✅ **Sophisticated Error Handling**
- ✅ **Support for Variables and Functions**
- ✅ **Support for Loops and Conditionals**

## 🏗️ Architecture

```
ProLang Mini-Compiler
├── Lexical Analysis (FLEX)
│   └── lexical.l → lex.yy.c
├── Syntactic Analysis (BISON)
│   └── syntax.y → syntax.tab.c & syntax.tab.h
├── Semantic Analysis
│   ├── ts.c / ts.h (Symbol Table)
│   ├── quadruplet.c / quadruplet.h (Intermediate Code)
│   └── asm.h (Assembly Code Generation)
└── Test Files
    ├── test.txt
    └── test2.txt
```

## 📦 Prerequisites

Before compiling the project, make sure you have:

- **GCC** (GNU Compiler Collection) or **MinGW** (for Windows)
- **FLEX** (Lexical Analyzer Generator)
- **BISON** (Parser Generator)
- **Make** (optional, for automating compilation)

### Prerequisites Installation

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get update
sudo apt-get install build-essential flex bison
```

**macOS:**
```bash
brew install flex bison gcc
```

**Windows (with MinGW):**
- Download [MinGW](https://www.mingw-w64.org/)
- Download [FLEX & BISON for Windows](https://gnuwin32.sourceforge.net/)
- Add paths to environment variables

## 🚀 Installation

1. **Clone the repository:**
```bash
git clone https://github.com/am11iin/ProLang--Mini-Compiler.git
cd ProLang--Mini-Compiler
```

2. **Compile the lexer (FLEX):**
```bash
flex lexical.l
```
This generates `lex.yy.c` from `lexical.l`

3. **Compile the parser (BISON):**
```bash
bison -d syntax.y
```
This generates `syntax.tab.c` and `syntax.tab.h` from `syntax.y`

4. **Compile the C files:**
```bash
gcc -o compil syntax.tab.c lex.yy.c ts.c quadruplet.c -lm
```

**Or in a single command:**
```bash
flex lexical.l && bison -d syntax.y && gcc -o compil syntax.tab.c lex.yy.c ts.c quadruplet.c -lm
```

## 📝 Technical Details

### Symbol Table (ts.c/ts.h)
Manages storage and retrieval of variable information:
- Variable name
- Type (int, float, etc.)
- Scope
- Memory address

### Quadruplets (quadruplet.c/quadruplet.h)
Intermediate representation of code:
- Operator
- Operand 1
- Operand 2
- Result

### Code Generation (asm.h)
Conversion of quadruplets to assembly code (x86 or other)

## 🐛 Error Handling

The compiler detects and reports:
- Lexical errors (invalid tokens)
- Syntactic errors (invalid structure)
- Semantic errors (incompatible types, undeclared variables)
- Scope errors

## ⚙️ Advanced Configuration

### Modifying the Grammar
Edit `syntax.y` to add new grammar rules
```bash
bison -d syntax.y
gcc -o compil syntax.tab.c lex.yy.c ts.c quadruplet.c -lm
```

### Adding New Tokens
Edit `lexical.l` to recognize new tokens
```bash
flex lexical.l
gcc -o compil syntax.tab.c lex.yy.c ts.c quadruplet.c -lm
```
